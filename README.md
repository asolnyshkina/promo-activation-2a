# Активация промокода — состояние ошибки (вариант 2a)

Живая интерактивная веб-страница по дизайн-хендоффу: экран **«Активация промокода»**
в личном кабинете TV-приложения (навигация с пульта, экранная клавиатура).

**[→ Открыть `index.html`](index.html)**

## Что это

Воспроизведение варианта **2a** из хендофф-пакета close-to-pixel по токенам и размерам.
Ключевое требование варианта: при появлении ошибки **поле и кнопка не сдвигаются**
(иначе фокус пульта «прыгает»), а сама ошибка — неброский текст прямо на серой карточке
под кнопкой.

Собрано с нуля по спецификации (`handoff/README.md`), а не распаковкой бандла.

## Интерактив

- **Три состояния** переключаются табами сверху: `Пусто` / `Ошибка` / `С клавиатурой`
  (соответствуют `handoff/screens/01–03-2a.png`).
- **Ввод промокода** — с физической клавиатуры или экранной ЙЦУКЕН (есть RU·EN).
- **Навигация «пультом»:** `←/→` открывают клавиатуру, `↑/↓` переводят фокус
  поле ↔ кнопка, `Enter` — применить.
- Неверный код → красная рамка поля + приглушённый текст «Неверный промокод»,
  **без сдвига вёрстки**. Верный код (`PROSENSE`) → успех в том же слоте.
- Экранная клавиатура выезжает под карточку и **не перекрывает кнопку «Применить»**.

## Отступления от буквы хендоффа

- Шрифт — системный стек вместо Manrope (в референсе он помечен как плейсхолдер).
- Success-состояние (верный код) добавлено для «живости» — в исходной спеке его нет.

## Структура

```
index.html                     — живая страница (самодостаточная, без внешних зависимостей)

handoff/screens/               — скриншоты трёх состояний варианта 2a
handoff/reference - all variants.html — оригинальный HTML-прототип со всеми вариантами
```
