# Handoff: Активация промокода — состояние ошибки (вариант 2a)

## Overview
Экран **«Активация промокода»** в личном кабинете TV‑приложения. Пользователь вводит промокод в поле и нажимает кнопку **«Применить»**. Если код неверный — показывается ошибка. Ключевое требование этого варианта: **кнопка «Применить» и поле ввода не должны сдвигаться при появлении ошибки** (иначе фокус пульта «прыгает»), а сама ошибка должна быть **неброской** и жить **прямо на серой карточке под кнопкой**.

## About the Design Files
Файлы в этом пакете — **дизайн‑референсы, сделанные в HTML** (прототипы, показывающие внешний вид и поведение), а **не продакшн‑код для копирования**. Задача — **воспроизвести этот экран в вашем кодовом окружении** (React / Vue / native TV framework и т.п.), используя ваши существующие компоненты, токены и паттерны фокусной навигации. Если готового окружения нет — выбрать подходящий фреймворк и реализовать экран в нём.

Это **TV‑интерфейс**: навигация с пульта (D‑pad), фокус, экранная клавиатура снизу. Учитывайте это при реализации фокуса и порядка обхода.

## Fidelity
**High‑fidelity.** Цвета, типографика, отступы и состояния — финальные, воспроизводить close‑to‑pixel в рамках ваших компонентов. Шрифт в референсе — **Manrope** (плейсхолдер); замените на системный/брендовый шрифт TV‑приложения, сохранив начертания и размеры.

## Screen: Активация промокода

Базовая система координат референса — экран **1120 × 630** (соотношение 16:9; масштабируйте пропорционально под реальное разрешение TV, обычно 1920×1080 — умножьте значения ниже на ~1.714).

### Layout
- **Фон экрана**: сплошной чёрный `#000000`.
- **Верхняя панель** (absolute, top 0): высота **74**, паддинг `0 42`.
  - Слева «‹ Назад» — цвет `#6E6E73`, 22px / 600.
  - «Личный кабинет» — белый `#FFFFFF`, 22px / 700, начинается на `left: 452`.
- **Сайдбар** (absolute, top 96, left 44): ширина **230**, вертикальный `flex`, `gap: 26`, размер пунктов 20px.
  - «Профили» — `#6E6E73` / 600
  - «Аккаунт», «Подписки», «Настройки», «Поддержка» — `#7C7C82` / 600
  - **Активный** «Активация промокода» — `#F6BE00` / 700
- **Карточка** (серая плашка, absolute, top 60, left 396): ширина **560**, паддинг `34 40`, фон `#262629`, радиус **20**, `flex column`, `box-sizing: border-box`. Карточка намеренно закреплена в **верхней зоне экрана**, чтобы экранная клавиатура (нижняя треть) её не перекрывала.

### Components внутри карточки (сверху вниз)
1. **Заголовок** «Активация промокода» — по центру, `#FFFFFF`, 30px / 800, `margin-bottom: 14`.
2. **Подзаголовок** — по центру, `#8B8B90`, 18px / 500, `line-height: 1.35`, `margin-bottom: 24`. Текст: *«Для некоторых сертификатов или промокодов может потребоваться привязать способ оплаты»*.
3. **Поле ввода**: высота **66**, радиус **12**, фон `#1B1B1E`, рамка **2px solid**, паддинг `0 22`, `gap: 14`.
   - Плейсхолдер «Промокод» — `#6E6E73`, 22px / 500.
   - Введённое значение — `#FFFFFF`, 22px / 600, `letter-spacing: .5`.
   - **Цвет рамки по состоянию**: в фокусе / обычное — `#F6BE00` (жёлтый); ошибка — `#F0616E` (красный).
4. **Кнопка «Применить»**: высота **60**, радиус **12**, `margin-top: 6` (кнопка прижата прямо к полю, без пустого отступа), текст 22px / 800.
   - **В фокусе (состояние ошибки)**: фон `#F6BE00`, текст `#1A1A1A`.
   - **Не в фокусе**: фон `#34343A`, текст `#C9C9CE`.
5. **Сообщение об ошибке** (только в состоянии ошибки): **под кнопкой, внутри серой карточки**, `margin-top: 16`, по центру, цвет **`#D98A90`** (приглушённый красный), 18px / 500. Текст: *«Неверный промокод»*. **Без плашки/заливки** — только текст, намеренно неброский.

## States (см. `screens/`)
- **`01-2a.png` — Ошибка**: поле заполнено (`FREESTART25`), рамка красная `#F0616E`, кнопка в фокусе (жёлтая), под кнопкой приглушённый текст «Неверный промокод». Поле и кнопка на тех же местах, что и без ошибки.
- **`02-2a.png` — Пусто**: плейсхолдер «Промокод», рамка жёлтая (поле в фокусе), кнопка не в фокусе (серая), ошибки нет.
- **`03-2a.png` — С клавиатурой**: открыта экранная клавиатура снизу, поле в фокусе (жёлтая рамка), идёт ввод (`FREESTART`), кнопка «Применить» **остаётся видимой над клавиатурой** — карточка не перекрыта.

## Interactions & Behavior
- Пользователь наводит фокус на поле → открывается экранная клавиатура (нижняя ~1/3 экрана) → вводит код.
- Фокус переходит на «Применить» → нажатие.
- Если код неверный — **под кнопкой** появляется приглушённый текст ошибки; поле подсвечивается красной рамкой; **ничего не сдвигается** (поле и кнопка стоят на месте, т.к. текст ошибки добавляется ниже кнопки и увеличивает карточку только вниз).
- Появление ошибки можно сопроводить лёгким fade‑in (150–200ms, ease‑out) — необязательно.
- **Клавиатура никогда не перекрывает кнопку**, потому что карточка закреплена в верхней зоне (top 60), а её нижний край выше зоны клавиатуры.

## State Management
- `value: string` — введённый промокод.
- `status: 'idle' | 'error'` — результат проверки (`error` включает красную рамку + текст ошибки).
- `focused: 'field' | 'button' | null` — что в фокусе пульта (управляет жёлтой рамкой поля и жёлтым фоном кнопки).
- `keyboardOpen: boolean` — открыта ли экранная клавиатура.
- Переход: нажатие «Применить» при неверном коде → `status = 'error'`. Изменение `value` → сброс `status` в `idle`.
